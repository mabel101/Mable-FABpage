As part of this process, I configured Git Bash, installed Visual Studio Code (VS Code), and connected my GitHub account. Process Documentation

Git and Gitbash

Git is a version control system that allows developers to track changes in their code, collaborate with others, and manage different versions of a project. Git Bash is a command-line tool that comes with Git for Windows, enabling users to run Git commands in a Linux-like environment.

Github

GitHub is an online platform built on top of Git that allows people to host, share, and collaborate on code or documentation projects. It serves as a cloud workspace for developers and teams around the world. In the context of the UniPod program, GitHub is our digital lab notebook - a space where we will upload our weekly tasks, tutorials, and reflections

Visual Studio Code (VSCode)

Visual Studio Code is a lightweight and user-friendly code editor developed by Microsoft. It supports multiple programming languages and integrates seamlessly with GitHub.

Setting up my my Github Page

To track my weekly progress in the UniPod Digital Fabrication Training, we used GitHub as our main documentation platform by creating individual GitHub Pages linked to the FabLab class repository. A GitHub Page is a website hosted directly from a GitHub repository. It allows users to publish and share their work online in a professional format. For our program, the page serves as an online portfolio where we record weekly tasks, reflections, and progress throughout the digital fabrication journey.

Step 1: Installing Git and Gitbash

I started by downloading Git for Windows and installing it on my computer. Once installed, I opened Git Bash to configure my username and email address, which identify my commits in Git. The commands I used were:

git config --global user.name "Mable Chileshe"

git config --global user.email "m.chileshe@alumni.alueducation.com"

Step 2: Setting up my Github Account

I logged into my existing GitHub account, which I had previously created during my computer science studies.

Step 3: Forking and Cloning the class repository

To get started, we were introduced to the FabLab class repository that contains the course structure and starter files. I began by forking the repository to create a personal copy under my own GitHub account. Forking means making a copy of an existing repository to be hosted on my own github. After forking, I cloned my copy of the repository to my local machine using Git Bash and Visual Studio Code (VS Code). This setup made it easier to edit files offline, commit changes, and push updates directly to GitHub. Cloning a repository means creating a local copy of an online coding project so that I can edit files on my computer while keeping them synchronized with GitHub. I used the following command in my VS Code terminal:

git clone

After cloning, I explored the folder structure and opened it in VS Code, where I could start editing my documentation.

Configuring My GitHub Page

Once the local setup was complete, I customized my GitHub Page by:

  • Updating my profile and repository details.
  • Editing the index.html and ‘About me’ files to reflect my information.
  • Organizing folders for weekly documentation and images..

Deploying the Page

To publish the page live as a github page:

  1. I opened my repository on GitHub.
  2. Went to Settings then Pages.
  3. Under Source, I selected the main branch and saved the configuration.
  4. GitHub automatically generated a link to my personal documentation site.
